# Presentation Slide 1 - Format Options
## Team Roles Section - Different Layout Options

---

## **Option 1: Concise Single-Line Format (RECOMMENDED)**
**Best for:** Fitting everything on one slide, easy to read

```
**Roles for Three Phases:**
- **D1 (Requirements)**: Andre (Coord), Alexandra (Requirements), Kamal (UC-003/004), Alfonso (UC-007)
- **D2 (Design)**: Andre (Architecture), Alexandra (Design Docs), Kamal (Seq Diagrams), Alfonso (All UML Diagrams)
- **D3 (Implementation)**: Andre (Ch. 5-6, Tests), Alexandra (Ch. 3/7, Tests), Kamal (Tests), Alfonso (App. D, Tests)
```

**Pros:**
- Very compact, fits easily on slide
- All information included
- Easy to scan

**Cons:**
- Uses abbreviations (D1, D2, D3, Ch., App.)

---

## **Option 2: Table Format**
**Best for:** Professional look, organized presentation

```
**Roles for Three Phases:**

| Deliverable | Andre | Alexandra | Kamal | Alfonso |
|-------------|-------|-----------|-------|---------|
| **D1 (Requirements)** | Coord, SRD | Requirements, Use Cases | UC-003/004 | UC-007 |
| **D2 (Design)** | Architecture | Design Docs | Seq Diagrams | All UML |
| **D3 (Implementation)** | Ch. 5-6, Tests | Ch. 3/7, Tests | Tests | App. D, Tests |
```

**Pros:**
- Very organized
- Easy to compare across deliverables
- Professional appearance

**Cons:**
- Takes more vertical space
- May need smaller font

---

## **Option 3: Grouped by Person**
**Best for:** Showing each person's contributions across all phases

```
**Roles for Three Phases:**
- **Andre (Team Lead)**: D1 (Coord), D2 (Architecture), D3 (Ch. 5-6, Tests)
- **Alexandra**: D1 (Requirements), D2 (Design Docs), D3 (Ch. 3/7, Tests)
- **Kamal**: D1 (UC-003/004), D2 (Seq Diagrams), D3 (Tests)
- **Alfonso**: D1 (UC-007), D2 (All UML), D3 (App. D, Tests)
```

**Pros:**
- Shows individual contributions clearly
- Compact format
- Easy to see who did what

**Cons:**
- Less emphasis on deliverables

---

## **Option 4: Minimal Version (Ultra-Compact)**
**Best for:** When space is very limited

```
**Roles:**
- **D1**: Andre (Coord), Alexandra (Requirements), Kamal (UC-003/004), Alfonso (UC-007)
- **D2**: Andre (Arch), Alexandra (Design), Kamal (Seq), Alfonso (UML)
- **D3**: Andre (Ch. 5-6), Alexandra (Ch. 3/7), Kamal (Tests), Alfonso (App. D)
```

**Pros:**
- Extremely compact
- Maximum space for other content

**Cons:**
- Very abbreviated
- Less descriptive

---

## **Option 5: Two-Column Layout**
**Best for:** Better use of slide width

```
**Roles for Three Phases:**

**Deliverable 1 (Requirements):**
- Andre: Project coordination, SRD oversight
- Alexandra: Requirements gathering, use case development
- Kamal: Use cases UC-003, UC-004
- Alfonso: Use case UC-007

**Deliverable 2 (Design):**
- Andre: Architecture design, design coordination
- Alexandra: Design documentation, design patterns
- Kamal: Sequence diagrams UC-003, UC-004
- Alfonso: All UML diagrams

**Deliverable 3 (Implementation):**
- Andre: Chapters 5-6, UC-001 & UC-005 test cases
- Alexandra: Chapter 3, Chapter 7, Appendix A, UC-002 & UC-006 test cases
- Kamal: UC-003 & UC-004 test cases
- Alfonso: Appendix D, UC-007 test cases
```

**Pros:**
- More detailed information
- Better readability
- Uses slide width effectively

**Cons:**
- Takes more space
- May need two slides or smaller font

---

## **Recommendation**

**For a standard presentation slide, I recommend Option 1 (Concise Single-Line Format)** because:
1. Fits comfortably on one slide
2. All information is included
3. Easy to read and understand
4. Professional appearance
5. Leaves room for other slide elements

**If you need more detail**, use **Option 5 (Two-Column Layout)** and split it across two slides if needed.

---

## **Tips for Fitting Content on Slides**

1. **Use abbreviations**: D1, D2, D3 instead of full deliverable names
2. **Shorten terms**: "Ch." for "Chapter", "App." for "Appendix", "Coord" for "Coordination"
3. **Remove redundancy**: "test cases" â†’ "Tests"
4. **Use smaller font**: 10-11pt for roles section if needed
5. **Consider splitting**: If too much content, use two slides (Slide 1a and 1b)
